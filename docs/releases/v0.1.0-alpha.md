# Rolecraft Job Agent `v0.1.0-alpha`

## Summary

`v0.1.0-alpha` is the first public alpha release of Rolecraft Job Agent.

It packages the project as a local-first, human-in-the-loop job application copilot that can:

- import jobs by URL
- analyze roles with a selectable LLM provider
- generate tailored resume versions and PDFs
- run best-effort Playwright prefills
- surface automation evidence for human review before submission

This is an alpha release for experimentation, local demos, and iterative improvement. It is not a promise of universal ATS coverage or production-grade multi-tenant deployment.

## Highlights

- Dual-provider LLM support with `OpenAI` and `Gemini`
- Structured job analysis and tailored resume generation
- Resume PDF preview and template switching
- Best-effort application prefill with screenshot and worker-log evidence
- Automation-session history and comparison for repeated prefill attempts
- Clear public setup docs, governance files, CI, and migration workflow

## What Is Included

### Product surfaces

- `Settings` for the active provider, model, and API key
- `Profile` for candidate details, experience, projects, and reusable default answers
- `Jobs` for imports, analysis, and resume generation
- `Resume Review` for tailored resume inspection and PDF export
- `Application Review` for prefill evidence, automation sessions, and approval state
- `Submission Review` for manual-submit tracking
- `Dashboard` and `Workflow runs` for cross-job history and orchestration visibility

### Runtime options

- `JOB_IMPORT_MODE=live|mock`
- `JOB_ANALYSIS_MODE=mock|live`
- `JOB_RESUME_MODE=mock|live`
- `TEMPORAL_ENABLED=true|false`

## Who This Alpha Is For

This release is most useful for:

- contributors who want a realistic local workflow to improve
- reviewers evaluating a transparent, human-in-the-loop alternative to "one-click apply" tooling
- developers who want a local-first demo of structured job analysis, resume tailoring, and best-effort application assistance

It is less suited for:

- users expecting production-grade hosted SaaS behavior
- teams needing multi-tenant auth and enterprise controls
- fully automated final submission to every ATS

## Known Limitations

- Final submit is still manual on purpose.
- Prefill is best-effort and will miss some site-specific or highly custom controls.
- The active LLM configuration is global: one provider, one model, one key at a time.
- Temporal support currently covers starter slices rather than a fully expanded workflow system.
- Demo screenshots and GIFs are planned, but this alpha ships with asset placeholders instead of final media.

## Setup Notes

The fastest local start remains:

```bash
cp .env.example .env
docker compose up --build
```

Optional demo data can be added after startup:

```bash
docker compose exec api npm run prisma:seed
```

For real deployments, set `JWT_SECRET` and `INTERNAL_API_TOKEN` explicitly and separately.

## Feedback Welcome

The most valuable alpha feedback is:

- gaps in the import -> analyze -> resume -> prefill -> review loop
- places where automation evidence is unclear or insufficient
- provider-specific issues with `OpenAI` or `Gemini`
- documentation or onboarding friction for first-time contributors

When reporting issues, please include whether you were using `mock` or `live` modes and which provider was active.
